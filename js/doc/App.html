App

new App()

An App is the controller for an entire application. It instantiates the top-level components of an application and connects them according to its configuration.

An App has three main parts to it:

Components

A component is any object in the application. It may be set up to communicate with other objects, and should follow the few restrictions of an App~Component type.

Events

Events are messages broadcast during the runtime of the application. An event can be sent by any component, and listened for by any component. The sender of the event may send arguments, which will be passed to all listeners.

Services

Services are functions from one component available for use by any other component.

Component

Properties:
Name Type Attributes Description
services Array | function <optional>
 The services the component will make use of, either as an array or a function that returns an array. Each service should either be an object with keys "name" and "source", or an array of size 2. "name", or the first element, should be a string representing the name to make the service available under. "source", or the second array element, should be a string representing the name of the service requested.
events Array | function <optional>
 The events the component will emit, either as an array or a function that returns an array. Each event should either be an object with keys "name" and "source", or an array of size 2. "name", or the first element, should be a string representing the name to give the event-emitting method. "source", or the second array element, should be a string representing the name of the event it should emit.
__app__ Object <optional>
 Settings that override the default names of component methods. Not normally needed, unless the component is already using properties "services" or "events" for something.

Type:

Properties:
Name Type Attributes Description
create string The path of the module with the class's constructor, which will be invoked to create the class.
params Array <optional>
 Parameters to be passed to the constructor. If only one non-array parameter is passed, it needn't be in an array.
data Object.<string, App~DataConfig> <optional>
 Data to put in as properties of the component after its creation, but before the app is started. Keys are property names, values specify what to assign to that property.

 Other components to connect this component to. The key is the property to assign the component reference to; the value is the name of the component. Because of the dependencies introduced by too many connections, this attribute should be used SPARINGLY.

DataConfig

Represents a single piece of data to be evaluated and used as a property of a component. A DataConfig object should have exactly one of the following keys -- it is an error to have none, and behavior is undefined if there is more than one.
Type:
  • Object
Properties:
Name Type Attributes Description

json string <optional>
 A JSON file to load and parse, the value of which will be the data to be used.
literal * <optional>
 A literal value like a number or a string, which will itself be the data to be used.
